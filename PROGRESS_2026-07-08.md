# mother-embedding — progress summary, 2026-07-08

## Goal

Rewrite the monolithic Pluto notebook `~/Julia/DATABASES/FPE_emnist_Gabor-pair_multiletter.jl`
(a VSA/Fractional-Power-Encoding letter classifier built on a Gabor-filter
front end) as a set of independent, individually testable Julia modules,
each `include`-able on its own (`include("X.jl"); using .X`), paired with
its own small Pluto notebook for visually sanity-checking that component
before moving to the next.

## Project setup

- Location: `/home/kevin/claude-code/mother-embedding`, a self-contained
  Julia project — its own `Project.toml`/`Manifest.toml`, independent of the
  user's shared `~/Julia` environment (deliberate, per request).
- Julia 1.11.2. Packages so far: Pluto, PlutoUI, Plots, Colors,
  ImageFiltering.
- EMNIST data is read in place from `~/Julia/DATABASES/EMNIST` (not copied
  into this project — the raw files are several GB).
- `run_pluto.sh` launches the notebook server matching the user's usual
  pattern: `Pluto.run(host="0.0.0.0", port=1235, launch_browser=false,

  browser over their own SSH tunnel to the lab server. **After editing any
  module or notebook file, the server must be restarted** (kill + relaunch
  `run_pluto.sh`) — a browser refresh alone does not pick up on-disk
  changes reliably.

## Components completed

### `Config.jl`
Single source of truth for project-wide constants, `include`d + `using`'d
before any component module. Currently holds the Gabor-lifting constants:
`IMG_SIZE`, `NATIVE_SIZE`, `SCALES`, `ORIENTATIONS`, `OVERLAP_FRAC`.
Referenced **bare** everywhere (`SCALES`, not `Config.SCALES`) — modules
and notebooks alike just `using .Config`. To change a value: edit
`Config.jl`, restart the Pluto server (these are `const`s — no live
mutation).

### `LoadEMNIST.jl` + `Test_LoadEMNIST.jl`
Parses the EMNIST balanced-train IDX files and buckets images by class.
- `load_emnist(; data_dir, n_images_to_load=20000, n_classes=5,
  class_order=default_class_order())` → named tuple `(images, labels,
  selected_classes, class_names, class_images)`.
- `to_image(m)` — fixes a display-orientation bug found during testing: raw
  EMNIST samples are already correctly oriented; the *display* needs a row
  flip (`reverse(dims=1)`) to render upright in `Plots.heatmap`, which
  defaults to bottom-up. (The original source notebook's `rotr90` approach
  was actually wrong for this purpose.)
- Test notebook: sliders for class count / images scanned, per-class image
  counts, a 5-image sample grid, and a per-class count bar chart.

### `CreateGaborLifting.jl` + `Test_CreateGaborLifting.jl`
Convolves an image with a bank of complex Gabor filters and samples the
response on a grid.
- `GaborSample`: `(x, y, θ, s, modulus, phase)` — carries the full complex
  response (modulus **and** phase), not just magnitude as in the original.
- `gabor_kernel(λ, θ)` — complex Gabor kernel, energy-normalized.
- `imresize_simple` — bilinear upscale 28×28 → 56×56 (kept as in the
  original; bicubic via ImageTransformations was considered but not used).
- `gabor_lift(image; scales, orientations, overlap_frac, img_size)` — raw,
  **unthresholded, unnormalized** output: every grid point at every
  scale/orientation is returned (the original applied a response threshold
  and dropped weak samples; this version doesn't). Boundary handling uses
  `Pad(:replicate)` (kept as in the original) — noted that at the largest
  scale the kernel radius is a large fraction of the image size, so most
  border tokens are influenced by replicated edge pixels.
- Simplified from the original's two-pass (separate real/imag `imfilter`)
  approach to a single complex-kernel `imfilter` call — verified numerically
  identical (~1e-7, float roundoff) and roughly halves the convolution work.
- `compute_response_grid` kept, commented out, for possible future use
  (superseded for normal use since `gabor_lift` itself now returns every
  grid point unthresholded).
- Test notebook: single-filter overlay (phase shown as line hue, modulus as
  line opacity), a modulus histogram, and a full scale×orientation panel
  grid for one image at a glance. Verified against known letter shapes
  (orientation-matched filters light up along matching strokes).
